## 프로젝트 개요

대학교 신입생 멘토링 프로그램 관리 시스템. Google Forms, Sheets, Calendar를 활용한 멘토 프로그램 등록 및 멘티 신청 관리 시스템입니다.

## 주요 프로젝트 컨텍스트

### 일정
- 멘토 등록: 2025년 9월 5일까지 (Google Forms 사용)
- 멘티 신청: 2025년 9월 15일 - 2025년 9월 19일까지
- 프로그램 운영: 2025년 9월 22일 - 12월 12일 (평일만)

### 비즈니스 요구사항
- 105명의 멘토가 3시간 프로그램 제공 (1시간 단위로 구성)
  - 1시간 짜리 3개 프로그램
  - 2시간 짜리 1개, 1시간 짜리 1개 또는 1시간 짜리 1개, 2시간 짜리 1개 
  - 3시간 짜리 1개
- 프로그램당 최대 7명 멘티
- 멘티당 최대 3개 프로그램 신청 가능
- 프로그램 간 시간 충돌 검사
- 모든 구현은 Google 생태계 내에서 진행

### 프로젝트 진행 요구사항
- 내가 직접 너가 알려준대로 하나씩 따라해야 하니 상세한 절차 가이드가 필요. 왜냐하면 내가 프로그래밍 초보자야.

## 아키텍처

### 데이터 레이어 (Google Sheets)
1. **멘토 프로그램 목록** (`result table.csv`의 Google Forms 응답)
   - 멘토가 제출한 프로그램 정보
   - 컬럼: result table.csv 참조

2. **신청현황 시트** (생성 필요)
   - 멘티 프로그램 신청 추적
   - 필수 컬럼: 신청ID, 멘티학번, 멘티이름, 멘티이메일, 프로그램ID, 신청일시, 상태

3. **설정 시트** (생성 필요)
   - 시스템 설정값
   - 신청 기간, 최대 신청 수, 정원 제한

### 비즈니스 로직 (Google Apps Script)
- 신청 유효성 검사 (멘티당 최대 3개, 중복 방지, 7명 정원)
- 프로그램 간 시간 충돌 감지
- Forms와 Sheets 간 데이터 동기화
- 자동 이메일 알림
- 확정 프로그램 캘린더 연동

### 프레젠테이션 레이어
- 멘티 신청 웹 인터페이스 (카드/테이블 뷰, 필터링 기능)
- 멘토용 신청자 조회 대시보드
- 관리자용 전체 통계 대시보드 (선택사항)

## 구현 우선순위

### Phase 1
- Forms 응답 데이터 구조 파싱
- 신청 관리 시트 생성
- 유효성 검사 포함 멘티 신청 웹앱 구축

### Phase 2
- 멘토 신청자 조회 구현
- 멘티 신청 상태 확인 추가
- 신청 마감 로직 처리

### Phase 3
- 공유 Google Calendar 생성
- 확정 일정 자동 등록
- 참가자 알림 이메일 발송

## 기술적 고려사항

- 모든 코드는 Google Apps Script 환경에서 실행 (JavaScript ES5/ES6)
- 배포는 Google Apps Script Web App 사용
- 데이터 작업은 Google Sheets API 활용
- 일정 관리는 Google Calendar API 사용
- 웹 인터페이스는 Google site 사용
